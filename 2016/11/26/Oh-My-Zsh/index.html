#Oh My Zsh

######Porque Z shell é mais legal que o seu shell (a menos que o seu shell seja zsh)

###Uma breve introdução à coisa toda

Se você é usuário linux novato provavelmente não sabe do que se trata o shell
e ainda está utilizando o que veio na sua máquina por default. Daí que você vê em
tutorias os cara tudo usando um terminal muito legal, todo colorido, cheio de
funcionalidades que o seu não tem e talz. Esse artigo vai tratar sobre isso, mais
ou menos…. leia.

###O que é Bash?! (pera, não estávamos falando de shell?)

Bash é o shell, ou interpretador de linguagem de comando, para os sistemas GNU. O nome
é um acrônimo para Bourne-Again SHell, um trocadilho em homenagem ao criador do shell
Stephen Bourne que o fez em 1977 nos Bell Labs. O bash é usado nas distribuições do GNU e
também utilizado no sistema operacional Linux

###Tá, agora me diz o que é Shell

O shell é simplesmente um macro processador que executa comandos. O termo macro processador
significa funcionalidade onde texto e símbolos são expandidos para criar expressões maiores

O shell é um interpretador de comandos e também uma linguagem de programação. Como um
interpretador de comandos, o shell provê uma interface de usuário para o conjunto de utilitários
do GNU. A linguagem de programação permite unir essas utilidades. Arquivos com comandos podem
ser criados e se tornar um conjunto de comandos. Shell pode ser interativo, com os comandos
enviados por meio do teclado para serem executados ou por meio da execução de um arquivo contendo
essas instruções.

Porque utilizar Zsh?

Com ele você pode ter funcionalidades como autocomplete para pastas, git e usar vários
plugins para customiza-lo.
